SSH and Security

Published on Dec 22, 2019
  • ssh
  • security
What is SSH ?

SSH stands for Secure Socket Shell or Secure Shell. It is a network protocol for accessing services sercurely over a unsecured network.

Why SSH ?
  • secure connection : ssh allows user to communicate to a remote server by encrypting entire connection, i.e. every packet send across the network is encrypted using a symmetric key cryptography
  • authentication : ssh use password based as well as assymetrical encryption based method to authenticate identity of client
  • executing commands : Also allows to execute commands on server shell and display output for commands on client screen.
Command
  • ssh -i <identity> <user>@<host> -p <port>
  • IdentityFile <identity> is the private key used for verifying client identity, a corresponding public key must be stored at serverâ€™s authorized_host to allow connection.
  • User <user> is the user account for which shell session should be created.
  • HostName <host> is the server address.
  • Port -p specify the port on which server is listening for incoming ssh requests.
